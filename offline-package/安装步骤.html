内部调查系统 · 离线安装指南

适用：Windows Server 2022（离线环境） | 包大小：~13MB

1 准备工作

包内容确认

确保安装包目录包含以下文件：

offline-package/
├── survey.exe          ← Go 后端 (纯静态, 无依赖)
├── config.json         ← 生产配置
├── install.bat         ← 一键安装脚本
├── setup-iis.ps1       ← IIS 反向代理脚本 (可选)
├── 安装步骤.html        ← 本文件
└── web/                ← 前端静态文件
    ├── index.html
    ├── logo.gif
    ├── css/
    └── js/

⚠ 安装前确认

  • 以 管理员 身份运行所有脚本和命令
  • 确保端口 8080 未被占用（或被防火墙阻止）
  • 如需 NTLM 域认证，请提前准备 IIS 扩展安装包（见 附录 A）
  • 如果想让其他机器访问本服务，确保 Windows 防火墙放行对应端口

2 快速安装（推荐）

  1. 将 offline-package 目录整个复制到服务器任意位置
  2. 右键 → 以管理员身份运行 install.bat
  3. 提示 "是否注册为 Windows 服务" 时，输入 y 回车

脚本自动完成：

  • 复制所有文件到 C:\SurveyServer\
  • 创建 C:\SurveyServer\data\ 数据目录
  • 注册并启动 Windows 服务 SurveyServer

3 手动安装

3.1 复制文件

mkdir C:\SurveyServer
mkdir C:\SurveyServer\data
xcopy /E /I web C:\SurveyServer\web
copy survey.exe C:\SurveyServer\
copy config.json C:\SurveyServer\

3.2 注册 Windows 服务

sc create "SurveyServer" binPath= "C:\SurveyServer\survey.exe" start= auto
sc description "SurveyServer" "内部调查系统"
sc start "SurveyServer"

验证服务状态：

sc query "SurveyServer"

3.3 验证运行

浏览器打开：

http://localhost:8080

看到问卷系统首页即表示安装成功。

如果无法访问，检查：

  • 服务是否启动 — sc query SurveyServer
  • 端口是否被防火墙阻止

4 配置说明

config.json 字段

字段默认值说明
port8080HTTP 监听端口
auth_mode"ntlm"认证模式: "ntlm" / "mock"
mock_username""mock 模式下固定用户名
initial_admin""首次启动时设为管理员的用户名
db_path"data/survey.json"数据文件路径（相对 exe 所在目录）

认证模式说明

mock 模式 不需要 IIS，适用于演示或无域环境：

{
  "auth_mode": "mock",
  "mock_username": "管理员",
  "initial_admin": "管理员"
}

ntlm 模式 需要 IIS 反向代理，适用于企业域环境：

{
  "auth_mode": "ntlm",
  "mock_username": "",
  "initial_admin": "DOMAIN\\your_account"
}

修改配置后需重启服务：

sc stop "SurveyServer" && sc start "SurveyServer"

5 IIS 反向代理（NTLM 认证必需）

仅 auth_mode: "ntlm" 时需要此步骤。 mock 模式可跳过。

5.1 安装 IIS 组件（离线可用）

以管理员身份运行 PowerShell，IIS 组件从系统 WinSxS 安装，不需联网：

Install-WindowsFeature -Name Web-Server, Web-Windows-Auth, Web-Mgmt-Console

5.2 安装 IIS 扩展（需提前下载）

这两个模块不在 Windows 系统内，需在有网机器提前下载：

模块下载地址安装
URL Rewrite 2.1 iis.net/downloads/microsoft/url-rewrite 双击 .msi 安装
ARR 3.0 iis.net/downloads/microsoft/application-request-routing 双击 .msi 安装

5.3 运行 IIS 配置脚本

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
C:\SurveyServer\setup-iis.ps1

脚本自动完成：

  • 创建 IIS 站点 SurveyProxy 监听 80 端口
  • 配置 Windows 认证（禁用匿名）
  • 设置反向代理规则到 http://localhost:8080
  • 转发用户身份头 X-Forwarded-User

6 防火墙配置

允许外部机器访问，放行端口：

# 直连模式（访问 8080）
netsh advfirewall firewall add rule name="Survey 8080" dir=in action=allow protocol=tcp localport=8080

# IIS 代理模式（访问 80）
netsh advfirewall firewall add rule name="Survey IIS 80" dir=in action=allow protocol=tcp localport=80

8 常见问题

Q: 服务启动后立即停止

查看日志 C:\SurveyServer\survey.log。通常是端口被占用或 config.json 格式错误。

Q: 浏览器访问被拒

检查 Windows 防火墙是否放行了端口。参见 第 6 步。

Q: NTLM 认证不工作

  • 确认 IIS 扩展（URL Rewrite + ARR）已安装
  • 确认 setup-iis.ps1 已运行
  • 浏览器访问必须用 http://服务器名（不能用 IP 或用 localhost），否则 Windows 认证不会触发
  • IE/Edge 默认启用集成认证；Chrome 需检查设置

Q: 如何修改端口

编辑 C:\SurveyServer\config.json 中的 port 字段，重启服务。

Q: 如何升级

  1. sc stop "SurveyServer"
  2. 替换 C:\SurveyServer\survey.exe 和 C:\SurveyServer\web\
  3. sc start "SurveyServer"
